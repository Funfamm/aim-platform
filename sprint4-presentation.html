10
💡 Reflections

What I Learned

Sprint 4 pushed into production infrastructure territory every challenge taught a principle I'll carry forward.

🚀

Pick the Right Platform for Your Stack

Render is a solid general platform, but Next.js has native framework-level integration with Vercel built-in ISR, edge functions, preview deploys. Lesson: Don't fight your framework. Move to where it's a first-class citizen.

🔐

Short-Lived Tokens Are Worth the Complexity

Implementing 15-minute token rotation added complexity silent refresh, token pair management, race conditions. But the security payoff is significant. Lesson: Security is not a feature you add later; it's architecture you design upfront.

☁️

Never Relay Files Through Your Server

The original upload pipeline sent files through the API route hitting Vercel's 4.5MB limit immediately. Presigned URLs let the client upload directly to storage. Lesson: Design data flow to move bytes the shortest path possible.

🌍

i18n Must Be Structural, Not Cosmetic

Bolting translations onto finished pages causes regressions. The deep-merge locale strategy ensures page-level keys never overwrite root keys. Lesson: Internationalization is an architecture decision plan the key structure before writing UI.

📊

If You Can't See It, You Can't Fix It

Without the email delivery log, failed emails were invisible. Without API key health badges, cooldowns were silent. Lesson: Observability isn't optional instrument everything from day one.

🗄️

Schema Migrations Need a Strategy

Using prisma db push vs prisma migrate works differently in dev vs production. Understanding when to use each prevents deployment failures. Lesson: Know your ORM's deployment model before you hit production.
