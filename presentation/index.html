Part 3

Payment Integrity Fix

A silent PayPal API limit was causing every donation to appear charged but never recorded.

🚨

The Root Cause — 127 Character Limit

PayPal's custom_id field (where donor metadata is stored) silently drops data above 127 characters. Our verbose JSON payload was ~155 characters, causing PayPal to truncate it. Result: payment charged, donation never recorded.

Critical Revenue Bug
✗ Before — 155 chars (over limit)
// JSON with verbose keys = 155+ chars
JSON.stringify({
  name: "Anonymous Donor",
  email: "samaderemi01@gmail.com",
  amount: 5,
  message: null,
  anonymous: true,
  userId: "clx9k2...",
  projectId: null
})
// PayPal silently drops → FAIL_SOFT
✓ After — ~95 chars (safe)
// Compact pipe-delimited format
[
  name.slice(0, 30),
  email.slice(0, 60),
  amount.toFixed(2),
  anonymous ? '1' : '0',
  userId || '',
  projectId || '',
].join('|')
// 100% reliably under 127 chars ✓
